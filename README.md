# ml_assignment_1

House Prices - Advanced Regression Techniques
კონკურსის მიმოხილვა:
ჩვენი მიზანია დავატრენინგოთ მოდელი, რომელიც დააფრედიქთებს თითოეული სახლის საბოლოო გასაყიდ ფასს Ames, Iowa-ში. მონაცემები მოიცავს 79 feature-ს, რომლებიც აღწერენ თითოუელ სახლს (მაგალითად, სამეზობლო, ავტოფარეხის ზომა, სარდაფი და ა.შ.). შეფასებისთვის Kaggle განსზაღვრავს RMSE-ის SalesPrice-ის ლოგარითმზე.

მიდგომა:
ჩემი მიზანი იყო, რომ დამტრენინგებინა შეძელბისდაგავარად საუკეთესო მოდელი. ჩემი მიდგომა იყო, რომ მეცადა ბევრი სხვადასხვა მიდგომა სხვადასხვა ეტაპზე (cleaning, feature engineering, feature selection, traiing) და გამეანალიზებინა, რამ იმუშავა, რამ არა, რატომ იმუშავა კონკრეტულმა მიდგომამ ან რატომ არ იმუშავა და ა.შ. შეფასების მეტრიკად Kaggle იყენებს RMSE-ს, ამიტომ, ცხადია, მე ყოველ ნაბიჯზე ვაკვირდებოდი მოდელის შედეგს rmse-ის საშუალებით. თუმცა, გარდა ამისა, ვიყენებდი R2 score-ს, რათა კარგად დამენახა სრული სურათი. გარდა ამისა, კონკრეტულ ნაბიჯებზე ვაკვირდებოდი plot-ებს და გამომქონდა შესაბამის დასკვნები.
Feature Engineering:
კატეგორიული ცვლადების რიცხვითში გადაყვანა:
Dataset-ში გვხვდებოდა 37 კატეგორიული სვეტი (მაგალითად, GarageType, Neighborhood და ა.შ.). ვინაიდან მოდელებს მხოლოდ რიცხვების გაგება შეუძლიათ, დამჭირდა მათი რიცხვითში გადაყვანა. 
პირველი მიდგომა, რომელიც გამოვიყენე იყო One Hot Encoding (pd.get_dummies(), drop_first=True). მას გადაჰყავს ყველა უნიკალური მნიშვნელობა სვეტში, შესაბამის 0/1 სვეტში. მაგალითად, ერთ-ერთი ასეთი იყო GarageType, რომლის მნიშვნელობებიც  (Attached, detached, no garage), გახდა 3 სვეტი. One hot encoding-ის გამოყენების შემდეგ ჩემი დატასეტი გაიზარდა 74 სვეტიდან 228-მდე.
ერთი პრობლემა რასაც წავაწყდი იყო რომ trian და test-ად დაყოფის შემდეგ, ზოგი კატეგორიული მნიშვნელობები გამოჩნდა train-ში, მაგრამ არ გამოჩნდა test-ში, რამაც გამოიწვია სხვადასხვა რაოდენობის სვეტები. ეს მოვაგვარე pd.align() და join=left-ის საშუალებით. ამის შემდეგ ორივეს ჰქონდა თანაბარი რაოდენობის სვეტები. ასევე შევამჩნიე, რომ სვეტები ბულიანებად გადაიქცა, ამიტომ ყველაფერი გადავაკეთე ინტებად.
One Hot Encoding ასევე გამოვიყენე cleaning-ის შედაგად მიღებულ სხვა დატაზეც (ამაზე ვისაუბრებ ქვევით), რამაც მომცა განცალკვებეული ენკოდირებული დატასეტი, რაზეც ასევე ჩავატარე ექსპერიმენტები.
Nan მნიშვნელობების დამუშავება:
იმის მერე რაც გავყავი მონაცემები train და test-ად, გადავამოწმე Nan მნიშვმელობები მხოლოდ train მონაცემებზე.
ამისთვის გამოვიყენე ორი მიდგომა.
1.	რიცხვითი სვეტებისთვის Nan მნიშვნლოებები შევავსე ამ სვეტის მედიანით (training set-ში). ხოლო კატეგორიული სვეტებისთვის გამოვიყენე მოდა.

2.	მეორე მიდგომაში იგივე გავაკეთე, ოღონდ რიცხვითებისთვის საშუალო გამოვიყენე მედიანის ნაცვლად. საშუალო უფრო სენსიტიური ექსტრემალური მნიშვნელობებისთვის. თუ სვეტს აქვს ძალიან დიდი მნიშვნელობები, საშუალო ძალიან გაიზრდება, მედიანა კი მაინც სტაბილური იქნება.

Cleaning მიდგომები:
Cleaning-ისთვის გამოვიყენე რამდენიმე საინტერესო მიდგომა.
პირველ რიგში შევამოწმე, რომელ სვეტებს ჰქონდა ბევრი Nan მნიშვნელობა. აღმოვაჩინე, რომ ზოგს ძალიან ბევრი გამოტოვებული მნიშვნელობა ჰქონდა. მაგალითად, PoolQC-ს ჰქონდა 99%, MiscFeature-ს 96% და ა.შ. ესე იგი, ამათ ისეთი ცოტა მნიშვნელობა ჰქონდათ, რომ მათ შევსებას, ცხადია, ვამჯობინე ამ სვეტების დადროპვა. გადავწყვიტე, რომ ისეთ სვეტებს, რომელთაც 40%-ზე მეტი მნიშვნელობა არ ჰქონდათ, გამეუქმებინა. ამან წამიშალა 6 სვეტი. დანარჩენი Nan მნიშვნელობებისთვის კი რიცხვითებისთვის გამოვიყენე მედიანა, კატეგორიულებისთვის მოდა. ამის შემდეგ დამრჩა 0 Nan მნიშვმელობა trian-შიც და test-შიც.
ამის შემდეგ, ვცადე მეორე მიდგომაც. პირველ რიგში, boxplot-ით გამოვსახე SalePrice. აქ ძალიან კარგად გამოჩნდა, რომ სახლების უმეტესობა ექცეოდა 100k-200k დაიპაზონში, მაგრამ ზოგი წერტილი იყო საკმაოდ დაშორებული. ამიტომ 500k-ს ზევით მდებარე სახლები წავშალე. ამან გამოიწვია 6 სტრიქონის დაკარგვა. აღვნიშნავ იმასაც, რომ ესენი წავშალე მხოლოდ ტესტ სეტში. შემდეგ გამოტოვებული მნიშვნელობები შევავსე საშუალოთი მედიანის ნაცვლად რიცხვითებისთვის, ხოლო კატეგორიულებისთვის მოდით.
ამ მიდგომების შედეგებზე (რომელმა რომელი მოდელი  გააუმჯობესა ან პირიქი გააურასე, ვისაუბრებ ქვემოთ).
Feature Selection
One hot encoding-ის შემდეგ 228 სვეტი მრჩება. ზოგი feature აქვს არამნიშვნელოვანი ინფორმაცია, ზოგი noise არის. გამოვიყენე რამდენიმე მიდგომა.
1.	პირველ შემთხვევაში, ყველა 228 სვეტი დავტოვე, რადგან მაინტერესებდა რა მოხდებოდა თუ feature selection-ს არ გმმოვიყენებდი. ასევე შევამოწმე კორელაცია feature-ების SalePrice-თან, რამაც მანახა ლოგიკური შედეგი.

2.	შემდეგ ასევე დამაინტერესა თუ რა შედეგს გამოიღებდა კორელაციის ფილტრი. გამოვითვალე ყველა feature-ის კორელაცია ერთმანეთთან. გადავწყვიტე, რომ თუ ორ feature-ს ექნებოდა 0.8-ზე მეტი კორელაცია ერთმანეთთან, ერთ-ერთი მათგანი დამედროპა, რადგან ისინი ერთნაირ ინფორმაციას ატარებდნენ და ორივეს დატოვება არ იყო საჭირო. მაგალითად , GarageArea და GarageCars ძალიან მაღალი კორელაციით გამოირჩეოდნენ, რაც ლოგიკურია. ეს გამოვსახე heatmap-ის საშუალებთ. საბოლოოდ, წავშალე 21 სვეტი. 

3.	შემდეგ გამოვიყენე RFE. RFE ატრენინგებს მოდელს, შლის გამოუსადეგარ feature-ებს და თავიდან ატრენინგებს. Base model-ად გამოვიყენე წრფივი რეგრესია და ამოვირჩიე 100 ყველაზე მნიშვნლოვანი feature. ამან დატასეტი 207-დან 100 სვეტამდე ჩამოიყვანა.

4.	RFE ასევე გამოვიყენე მეორე cleaned dataset-ზე (ანუ იმაზე, რომელშიც აუთლაიერები წაშლილი მქონდა და mean imputation გავაკეთე), რათა მენახა ეს უკეთ იმუშავებდა თუ არა

ამ მიდგომების შედეგებს ნახავთ ქვემოთ.


Training
Linear Regression:
თავიდან დავიწყე წრფივი რეგრესიით. გავტესტე ყველაფერზე და დავინახე feature selection-ის ეფექტი.
შედეგები შემდეგია:
პირველ რიგში გავტესტე ყველა სვეტზე, ესე იგი, feature selection-ის გარეშე. შედეგები მოსალოდნელი იყო. შემდეგ გამოვიყენე კორელაცია, შემდეგ RFE და შემდეგ RFE აუთლაირების გარეშე. 
ამან მაჩვენა ძალიან დიდი overfit ყველა feature selection-ის შემთხვევაში. აღმოჩნდა, რომ ყველა შემთხვევაში Train R2 იყო ძალიან მაღალი, მერყეობდა 0.89-0.94 დიაპაზონში. ეს ნიშნავს, რომ მოდელი ტრნინგ დატას ძალიან კარგად ერგება, მაგრამ სამაგიეროდ Test R2-მა მაჩვენა ძალიან ცუდი შედეგი. Test R2 მიჩვენებდა ნეგატიურ რიცხვებს. მაგალითად, Feature selection-ის გარეშე აჩვენა -39.06, კორელაციით აჩვენა -36.63. ჩემი დაკვირვებით, ეს მოხდა იმიტომ, რომ იყო ბევრი feature და შედარებით ცოტა სტრიქონი. წრფივმა რეგრესიამ უბრალოდ დაიმახსოვრა ტრენინგ დატა. 
RFE-მ საკმაოდ გააუმჯობესა მოდელი. Test R2 ჩამოვარდა -5.26-ზე და TestRMSE 2.73-დან 1.08-ზე. თუმცა, ცხადია, მოდელს მაინც ძალიან დიდი overfit ჰქონდა, რაც მოსალოდნელი იყო (თუმცა ასეთ დაბალ რიცხვებს არ ველოდი). აუთლაიერების წაშლამ კი ცოტათი გააუმჯობესა შედეგი, Test RMSE 1.08-დან 1.04-ზე ჩამოვიდა.
